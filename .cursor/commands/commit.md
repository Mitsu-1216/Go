description: 変更を確認し、適切なメッセージでコミットする
---

## 目的

作業ツリーの変更を確認し、意味のある単位でステージしてコミットする。

## 手順

1. `git status` と `git diff` で変更内容を把握する。
2. コミットに含めるべきでないファイル（生成物、秘密情報、ローカル設定など）がステージされていないか確認する。
3. 変更の意図に沿ったコミットメッセージを付ける（Conventional Commits やプロジェクトの慣習があればそれに従う）。
4. ユーザーが明示的に求めていない限り、`git push` は行わない。

必要ならユーザーにコミットメッセージ案を提示してから実行する。
